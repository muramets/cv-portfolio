# Vibe Player — Easter Egg Пасхалка с треком в воротах Impact

## Зачем существует эта документация

Ворота Impact на странице `about.html` содержат встроенную кнопку-пасхалку «EXPERIENCE WITH MUSIC». При клике на эту кнопку запускается фоновый музыкальный трек («Jesus Walks»), а сама кнопка плавно анимируется и передаёт управление компактному плееру в углу экрана.

Этот документ описывает бизнес-логику работы плеера, поведение анимации и адаптивность под мобильные браузеры (iOS Safari / iPad).

---

## Текущее состояние

← YOU ARE HERE (Стабильная версия)

- **Аудио**: Загружается по требованию при появлении секции ворот в поле зрения. Проигрывается с мягким нарастанием громкости (fade-in) и плавным затуханием (fade-out) в конце.
- **Громкость**: Полностью управляемая на всех устройствах, включая iOS Safari (iPad / iPhone), где стандартные HTML5 volume-контролы игнорируются системой.
- **Анимация ворот**: Кнопка запуска анимируется внутри собственного контейнера, сохраняя 100% синхронизацию с анимацией скролла створок ворот ("01" / IMPACT).

## Technical Implementation

### Файлы и структура
- `js/features/vibe-player.js`: Логика загрузки звука через Web Audio API, управление узлом `GainNode`, анимация выравнивания надписи `"01"` и переключение на угол экрана.
- `about.html`: Разметка кнопки `.vibe-gate-btn` с обёрткой `.vibe-gate-btn__inner`.
- `css/layout.css`: Стили ворот, кнопки `.vibe-gate-btn` и контейнера `.vibe-gate-btn__inner`.

### Особенности работы с Web Audio на iOS
- `HTMLMediaElement.volume` в iOS Safari игнорируется операционной системой. Для точной регулировки громкости используется `AudioContext` + `AudioBufferSourceNode` + `GainNode`.
- Внешняя кнопка `.vibe-gate-btn` сохраняет свой CSS-scroll-timeline (`impact-shutter-right`), а анимация увеличения при клике применяется к внутреннему контейнеру `.vibe-gate-btn__inner`.
